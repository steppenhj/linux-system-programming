tty: 내가 현재 접속해서 명령어를 치고 있는 창(터미널)의 Device file 경로를 알려주는 명렁어.
stty: 터미널의 성격을 뜯어고치는 명령어. 

fcntl: 권한을 다루는 것이 아니라, 파일이 현재 어떻게 작동하고 있는지 상태(status flag)를 제어하는 시스템 콜

## 🗂️ Disk File vs. Device File 비교

리눅스/유닉스 시스템에서는 장치(Device)도 파일처럼 취급합니다. 디스크 파일과 장치 파일은 겉보기엔 비슷하게 동작하지만, 내부 메커니즘과 제어 방식에서 명확한 차이를 가집니다.

### 🤝 공통점 (Similarities)

* **기본 파일 속성 보유**
  일반 파일처럼 장치 파일도 파일명, 아이노드(Inode) 번호, 소유자, 접근 권한 비트(Permission bit), 마지막 수정 시간 등의 속성을 똑같이 가집니다.
* **동일한 시스템 콜 사용**
  데이터를 읽고 쓸 때 `open`, `read`, `write`, `close`, `lseek` 같은 표준 입출력 함수들을 디스크 파일과 장치 파일 양쪽에 모두 동일하게 사용할 수 있습니다.
* **보안 및 접근 제어**
  일반 파일을 보호하는 것과 완전히 동일하게, 파일 권한 비트를 통해 장치에 대한 접근 권한을 제어합니다.

### ⚖️ 차이점 (Differences)

| 비교 항목 | 💾 Disk File | 🖥️ Device File |
| :--- | :--- | :--- |
| **아이노드(Inode) 지시 대상** | 디스크 물리적 공간에 있는 실제 **데이터 블록(Data Block)**을 가리킴 | 데이터 블록이 아닌, 해당 장치를 제어하는 커널 내부의 **장치 드라이버(Device Driver)**를 가리킴 |
| **속성 및 데이터 처리 방식** | 주로 단순한 파일 시스템 **버퍼링(Buffering)**과 관련된 속성을 다룸 | 통신 전송 속도(Baud rate), 화면 에코(Echo), 편집 기능, 줄바꿈 변환 등 **복잡한 연결 속성**을 가짐.<br>*(데이터 단순 전송이 아닌 드라이버를 통한 가공 및 처리 발생)* |
| **설정 제어(Control) 방법** | `fcntl` 시스템 콜을 사용하여 디스크 파일 연결 설정(`O_SYNC`, `O_APPEND` 등)을 제어 | `ioctl` 시스템 콜을 사용하거나, 터미널의 경우 `stty` 명령어(`tcgetattr`, `tcsetattr` 시스템 콜)를 사용하여 드라이버 세부 설정 제어 |
