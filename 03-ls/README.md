ls 및 stat 관련해서 배운 week 3

1) 어떤 파일의 stat 구조체 내 st_mode를 확인한 결과, 권한이 rwxr----x로 설정.
- 이 파일의 접근 권한을 8진수
=> 741

-시스템(커널) 내부에서 이 파일에 '사용자 읽기(User-read)' 권한이 있는지 확인하기 위해 어떤 기법과 상수를 사용하는지 설명
=> 커널은 비트 마스크 기법을 사용하여 특정 비트가 설정(set)되었는지 확인함. 사용자 읽기 권한을 확인하기 위해 S_IRSUR (0400) 마스크를 사용하여 st_mode 값과 비트 논리곱(AND) 연산을 수행.


2) ls 명령어는 디렉토리 내의 파일 목록을 보여주고, ls -l은 상세 속성까지 보여준다. 커널 구조도를 참고하여 
- 디렉토리 내용을 읽어오는 시스템 콜과
=> readdir(): 커널이 디렉토리 블록에 접근하여 그 안의 파일 이름(엔트리)들을 순차적으로 읽어올 때 사용하는 시스템 콜

- 파일의 상세 속성을 읽어오는 시스템 콜을 각각 설명
=> stat(): 커널이 일반 파일이나 디렉토리의 속성(properties) 데이터 블록에 접근하여 크기, 권한, 링크 수 등의 메타데이터를 가져올 때 사용하는 시스템 콜.
